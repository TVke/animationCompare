Syntax

The CSS syntax is simple and easy to write. It is part of CSS 3 and is largely supported in modern browsers. The control over timing is a very nice extra, in this example it is set to linear but ease-in, ease-out and ease are commonly used. It is even possible to set a cubic bezier to manage time.

Syntax

The JS syntax is complex and mathematical harder to write. Of course there are libraries to make it easier like GSAP and Velocity.js (two of the fastest libraries for JS animations)

@keyframes <name>{
  from{left:0;}
  to{left:100%;}
}

.ball{
  animate: <name> 3s linear infinite;
}
var counter = 0, step = (100 - 0) / (3 / (1 / 50));
var animate = setInterval( function(){
  if( 0 + (counter * step) <= 100 ){
    ball.setAttribute("style", "left:" + roundNumber(0 + (counter * step), 2) + "%");
    ++counter;
  }else{
    ball.setAttribute("style", "left:" + 100 + "%";
    clearInterval(animate);
  }
}

Conclusion

CSS and JS animations are both meaningful.

Just use CSS for the simple stuff and use JS for the events and random parts.

If you have the feeling that you need a lot of unnecessary calculations, try a JS animation library like GSAP(Green Sock) or Velocity.js.

One other thing JS is utilizing the CPU and CSS is using the GPU to make the animations happen so together they use both and not only one.
